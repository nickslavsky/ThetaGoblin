# IV30 & IVRank Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Capture a weekly IV30 reading per symbol from yfinance, build a historical time series, compute IVRank when enough history exists, and surface IV rank in the candidates UI — all gracefully degrading when data is young.

**Architecture:** Add a new `IV30Snapshot` model to store the weekly IV30 time series (one row per symbol per week). Extend `yfinance_svc` with a `get_iv30()` function that reads `Ticker.info["impliedVolatility"]`. Add a `compute_iv_rank` management command that (a) pulls today's IV30 for each qualifying symbol, (b) computes rank from the historical IV30 series, and (c) writes an IVRank row. Wire this into `run_options_pipeline`. The candidates view and template gain an IV rank badge per ticker. No failures, no exceptions when history is thin — just `is_reliable=False` and a visual cue.

**Tech Stack:** Python 3.12, Django 5.x, yfinance, PostgreSQL 17

---

## Definition of Done

1. Every weekly options pipeline run stores one IV30 value per qualifying symbol.
2. IVRank is computed and stored for every symbol that has ≥2 IV30 data points (min/max requires at least 2). Symbols with <2 data points get no IVRank row — they are simply too new.
3. `is_reliable=True` when `weeks_of_history >= 52`. Otherwise `is_reliable=False`.
4. `iv_percentile` is computed alongside rank (percentile = % of historical readings ≤ current).
5. Candidates view shows IV rank per ticker when available, with a visual flag when unreliable.
6. Candidate filtering by `iv_rank_min`/`iv_rank_max` applies only to symbols with reliable IV rank. Unreliable symbols pass the filter (benefit of the doubt — we'd rather show a candidate with thin IV data than silently drop it).
7. No command crashes on first run (zero history), second run (one data point), or any subsequent run.
8. All new code has tests. Full suite passes.

## Success Criteria

- After first `run_options_pipeline`: IV30Snapshot rows exist, IVRank table still empty (need ≥2 readings).
- After second weekly run: IVRank rows exist with `is_reliable=False`, `weeks_of_history=2`.
- After 52 weekly runs: IVRank rows show `is_reliable=True`, `weeks_of_history=52`.
- Candidates UI shows "IV Rank: 74" or "IV Rank: N/A (building)" per ticker.
- Dropping `iv_rank_min`/`iv_rank_max` FilterConfig thresholds to 0/100 effectively disables the filter.

class IV30Snapshot(models.Model):
    """Weekly IV30 reading from yfinance. One row per symbol per snapshot date."""
    symbol = models.ForeignKey(Symbol, on_delete=models.CASCADE, related_name="iv30_snapshots")
    date = models.DateField()
    iv30 = models.FloatField(help_text="30-day implied volatility (decimal, e.g. 0.28)")

    class Meta:
        unique_together = ["symbol", "date"]
        ordering = ["-date"]
        indexes = [
            models.Index(fields=["symbol", "date"]),
        ]

    def __str__(self):
        return f"{self.symbol.ticker} {self.date} IV30={self.iv30}"

def get_iv30(ticker: str) -> float | None:
    """Fetch 30-day implied volatility from Yahoo Finance.
    Returns the decimal value (e.g. 0.28) or None if unavailable.
    """
    try:
        t = yf.Ticker(ticker)
        val = t.info.get("impliedVolatility")
        if val is None:
            return None
        f = float(val)
        if math.isnan(f):
            return None
        return f
    except Exception:
        logger.exception("Failed to get IV30 for %s", ticker)
        return None
```

### Task 3: IV Rank Computation Service

**Context:** Pure computation module — takes the current IV30 and a historical series, returns rank and percentile. No ORM, no network. This is the math that the management command will call.

The formulas:
- **IV Rank:** `(current - min) / (max - min) * 100`. Undefined (return None) when max == min or fewer than 2 data points.
- **IV Percentile:** `count(historical_values <= current) / total_count * 100`. Requires ≥2 data points.
- **is_reliable:** `weeks_of_history >= 52`

RELIABLE_WEEKS = 52


def compute_iv_rank(
    current_iv30: float,
    historical_iv30s: list[float],
) -> dict | None:
    """Compute IV rank, percentile, and reliability from a historical IV30 series.

    Args:
        current_iv30: Today's IV30 reading.
        historical_iv30s: All historical IV30 values (including current).

    Returns:
        Dict with iv_rank, iv_percentile, weeks_of_history, is_reliable.
        Returns None if there are fewer than 2 data points or min == max.
    """
    n = len(historical_iv30s)
    if n < 2:
        return None

    iv_min = min(historical_iv30s)
    iv_max = max(historical_iv30s)

    if iv_max == iv_min:
        return None

    iv_rank = (current_iv30 - iv_min) / (iv_max - iv_min) * 100
    iv_percentile = sum(1 for v in historical_iv30s if v <= current_iv30) / n * 100

    return {
        "iv_rank": iv_rank,
        "iv_percentile": iv_percentile,
        "weeks_of_history": n,
        "is_reliable": n >= RELIABLE_WEEKS,
    }
```

```python
import logging
import time
from datetime import date

from django.core.management.base import BaseCommand

from screener.models import IV30Snapshot, IVRank
from screener.services import yfinance_svc
from screener.services.candidates import get_qualifying_symbols
from screener.services.iv_rank_svc import compute_iv_rank

logger = logging.getLogger(__name__)


class Command(BaseCommand):
    help = "Pull IV30 for qualifying symbols, store snapshot, compute IV rank"

    def add_arguments(self, parser):
        parser.add_argument(
            "--delay", type=float, default=0.5,
            help="Seconds between yfinance calls (default: 0.5)",
        )

    def handle(self, *args, **options):
        delay = options["delay"]
        today = date.today()
        symbols = get_qualifying_symbols()

        self.stdout.write(f"Computing IV rank for {len(symbols)} qualifying symbols...")

        iv30_stored = 0
        ranks_stored = 0
        skipped = 0

        for sym in symbols:
            iv30 = yfinance_svc.get_iv30(sym.ticker)
            if iv30 is None:
                logger.warning("No IV30 available for %s, skipping", sym.ticker)
                skipped += 1
                if delay > 0:
                    time.sleep(delay)
                continue

            IV30Snapshot.objects.update_or_create(
                symbol=sym,
                date=today,
                defaults={"iv30": iv30},
            )
            iv30_stored += 1

            # Load full history for this symbol
            history = list(
                IV30Snapshot.objects.filter(symbol=sym)
                .order_by("date")
                .values_list("iv30", flat=True)
            )

            result = compute_iv_rank(
                current_iv30=iv30,
                historical_iv30s=history,
            )

            if result is not None:
                IVRank.objects.update_or_create(
                    symbol=sym,
                    computed_date=today,
                    defaults={
                        "iv_rank": result["iv_rank"],
                        "iv_percentile": result["iv_percentile"],
                        "weeks_of_history": result["weeks_of_history"],
                        "is_reliable": result["is_reliable"],
                    },
                )
                ranks_stored += 1

            if delay > 0:
                time.sleep(delay)

        self.stdout.write(
            self.style.SUCCESS(
                f"Done. IV30 stored: {iv30_stored}, ranks computed: {ranks_stored}, "
                f"skipped: {skipped}"
            )
        )
```

## Notes for the Implementer

**On the yfinance `impliedVolatility` field:** This is Yahoo Finance's own computation. It uses both calls and puts, weighted around ATM, and interpolated to 30 days. It's not documented precisely, but it's consistent enough to rank a symbol against its own history — which is all IV rank needs.

**On the `is_reliable` / benefit-of-the-doubt policy:** This is a product decision, not a technical one. A new symbol with 3 weeks of IV history might have a "rank" of 95 that's meaningless — it's comparing against 3 data points. We don't want to block it or highlight it as a strong candidate. We show it with an asterisk so the user knows the data is thin and can use judgement.

**On running the command manually before 52 weeks of history:** You can always `docker compose exec web python manage.py compute_iv_rank --delay=0.5` to pull IV30 snapshots and compute rank. The rank will be unreliable but present. The pipeline does this automatically on Fridays. After 52 weekly runs (about a year), the ranks become reliable.

**On `iv_percentile` vs `iv_rank`:** IV rank uses min/max, so a single extreme reading can skew it. IV percentile is more robust — "74th percentile" means 74% of historical readings were at or below the current level. Both are stored. The view currently displays rank (which is what most options screeners show), but you could switch to percentile later without re-fetching data.
